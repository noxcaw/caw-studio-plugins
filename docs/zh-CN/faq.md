# 常见问题

## 我必须写 Python 吗？

不。只写声明式贡献（芯片支持包 / 命令菜单文案 / 设置 / i18n）而不写 handler 的插件完全不需要
Python。只有当插件声明了 `python` 入口且用户去触发命令/任务/烧录时才启动宿主进程。当然，命令
的 `handler` 若无 Python 入口，点击会提示"需要运行时"。

## 开发时每次都要发布吗？

不。用**侧载**：把含 `manifest.json` 的目录拷进 `~/.caw-studio/plugins/<目录名>/`，插件面板刷新
即生效。改 manifest 刷新、改 Python 重新调用命令即可。详见[创建第一个插件](quickstart.md)。

## 侧载后插件不出现？

- 确认目录里有 `manifest.json`，且 `id` 合法（`^[a-z0-9][a-z0-9-]{1,63}$`）。
- 在插件面板点刷新。
- manifest 有解析错误时插件详情页会提示——已知贡献点写错（缺必填字段、类型不对）是硬报错。
- 与注册表已装的**同 id** 冲突时注册表优先，换个 id 或先卸载注册表版本。

## `${chip}` / `${tool:ID}` 报错？

- `${chip}`：项目没有 `.ioc`（拿不到 Mcu 名）时该变量不可用。用芯片支持包的
  `match.project` 探测非 CubeMX 项目，或在步骤里避免依赖 `${chip}`。
- `${tool:ID}`：`ID` 必须是某制品 `tools[].id` 里声明过的 id，且该工具制品已安装。侧载纯声明
  插件本身不带工具时，`${tool:...}` 自然解析不到——它靠另一个工具链插件提供。

## 为什么我改了 Python 代码没生效？

宿主按调用懒启动、执行完不常驻，通常重新调用命令就会跑新代码。若宿主仍活着（比如 webview
面板长开占着进程），**停用再启用插件**强制重启宿主。

## `ctx.settings` 里 bool 设置读出来是字符串？

是的，`ctx.settings` 的值都是**字符串**。`bool` 型设置读到的是 `"true"`/`"false"`，判断写
`ctx.settings.get("my.loud") == "true"`。

## 发布时 CI 说版本不一致？

发布器要求 tag 去掉前导 `v` 后与 `manifest.json` 的 `version` **严格一致**（`X.Y.Z`）。先改
manifest 的 `version` 并提交，再打同号 tag。这是防"tag 与内容漂移发错版"的刻意设计。

## 怎么发 beta？

打带 `-beta` 后缀的 tag（`v0.2.0-beta`）。客户端插件页开"Beta 渠道"开关的用户才看得到。稳定后
再发同版号的 stable tag。见[版本与兼容性](versioning.md#beta-渠道)。

## 重发同一版本会重复上传制品吗？

不会。发布器打**确定性 tar.gz**，同内容同 `sha256`，注册表内容寻址识别为已有制品，秒传跳过
上传。

## 我能自助发布社区插件吗？

当前不行。发布仅限官方（`publish_plugin` scope 的 token）。制品签名、`python.permissions` 强制
执行、社区 tier 放开都在路线图上。开发/自用可以一直用侧载。

## 插件能访问设备或文件系统吗？

Python 宿主进程本身能（它就是普通 Python 进程），但 `python.permissions` 字段目前只存储不
强制——预留给社区发布上线。webview 面板则被沙箱隔离，**不能**直接碰设备/文件系统，一切副作用
必须经声明过的命令走宿主。

## 卸载插件会删掉共享工具吗？

只删该插件独占的东西。工具制品在共享池里按内容寻址，被其它已装插件引用的会保留；不足 1 小时的
新鲜目录也豁免（防误删并发安装）。停用只翻登记位、制品全留。

## 升级插件会丢我的设置吗？

不会。插件设置存 `plugins/<id>/settings.json`，与版本目录平级，升级只换版本目录，设置、
启用/停用与 `auto_update` 选择都保留。
