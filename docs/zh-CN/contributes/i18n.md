# 语言包（i18n）

`contributes.i18n` 声明各语言的语言包文件；manifest 里所有 `%key%` 占位文案展示时经它解析。

```jsonc
"i18n": {
  "en": "i18n/en.json",
  "zh-CN": "i18n/zh-CN.json"
}
```

- 键 = locale（目前支持 `en`、`zh-CN`），值 = 语言包 JSON 文件路径（相对插件 files 根）。
- 只声明的 locale 会被加载。

## 语言包文件格式

平面 JSON：键 → 文案。键就是 manifest 里 `%key%` 去掉百分号的名字。

`i18n/zh-CN.json`：

```json
{
  "cmd.sayHello": "打个招呼",
  "set.greeting": "问候用语",
  "set.greeting.desc": "打招呼时使用的词",
  "set.loud": "大声问候",
  "panel.hello": "问候面板"
}
```

`i18n/en.json`：

```json
{
  "cmd.sayHello": "Say Hello",
  "set.greeting": "Greeting word",
  "set.greeting.desc": "Word used when greeting",
  "set.loud": "Shout the greeting",
  "panel.hello": "Greeting Panel"
}
```

## `%key%` 用在哪

命令、设置、面板的展示文案支持 `%key%` 占位：

- `command.title`
- `setting.title` / `setting.description`
- `panel.title`

例如 `"title": "%cmd.sayHello%"` 展示时替换为当前语言下 `cmd.sayHello` 的文案。

## 运行时行为

- 语言包在运行时挂进前端 **overlay**，键空间为 `plugin.<id>.*`（各插件互不冲突）。
- **查找链**：当前语言 → `en` → 键名原样。即某键缺当前语言翻译时回退 `en`，再缺则显示键名
  本身（便于开发期发现漏翻）。
- 应用通过 `plugin_i18n_read` 读取语言包 JSON。

## 建议

- 保持 `en` 完整——它是回退语言，任何键都应至少有英文。
- 键名用点分层次（`cmd.*` / `set.*` / `panel.*`）便于组织，但这只是惯例，键空间是平面的。
